# BaoJunKeyless 无感状态机与后台预唤醒设计

## 目标

- 自动解锁只发生在“真实离开后首次重新靠近”的边沿。
- RSSI 分为近场、灰区、离开；灰区不触发锁解。
- 信号丢失不是离开，不得自动上锁。
- 每次判定记录统一安全快照。
- BLE ACK 本地即时反馈，HTTP 原始完整回包最终确认。
- 围栏和显著位置变化只预唤醒 BLE，绝不直接锁解。

## RSSI 状态机

```text
近场：rssi >= unlockThreshold
灰区：lockThreshold < rssi < unlockThreshold
离开：rssi <= lockThreshold
```

- 近场：只有此前观察到真实离开，才生成一次 `approachEdge`。
- 灰区：取消连续弱信号，不解锁，不开始上锁倒计时。
- 离开：设置已离开证据，开始连续弱信号计时。
- RSSI 丢失：清除离开倒计时，不上锁。

## 自动解锁

必须同时满足：

- 无感与自动解锁开启；
- BLE 已完成鉴权；
- RSSI 当前为近场；
- 本会话已观察到真实离开，并首次重新靠近；
- 车辆锁态和档位等安全条件允许；
- 无外部锁车保护、负 ACK 保护、手动反向抑制；
- 当前没有其他控制命令。

边沿只能在 BLE 已就绪且真正下发命令前消费。BLE 未鉴权时不提前消费。

## 自动上锁

必须同时满足：

- 无感与自动上锁开启；
- 本会话曾进入近场；
- BLE 已鉴权；
- 当前 RSSI 为离开区，且有 live 弱 RSSI；
- 持续弱信号达到 `lockDelay`；
- 车辆未锁、车速/档位/电源等安全条件允许；
- “上锁前检查门尾”开启时，门与尾门明确关闭；车窗不阻断锁车。

锁车成功后约 2.1 秒请求 HTTP 完整车况。最终锁态直接读取**本次新 HTTP 原始 `doorLockStatus`**，不读取 UI 或受 BLE 本地保护的 `state.locked`。门、尾门、车窗任一明确未关都会提醒。

## 控制后状态机

```text
BLE ACK 成功
→ 本地即时写入锁态
→ 请求 HTTP 原始完整快照
→ HTTP 原始 doorLockStatus 最终确认

BLE 负 ACK
→ 立即报告车辆拒绝
→ 本次靠近周期不再重试
→ 真实离开并重新靠近后才恢复

外部锁态 未锁→已锁
→ 开启外部锁车保护
→ 手机仍在近场不自动解锁
→ 真实离开并重新靠近后解除
```

## 统一安全快照日志

每次解锁、启动电源、上锁允许/等待/拒绝记录：

```text
locked / doors / windows / trunk / gear / power
httpAge / bleAuth / rssi / zone / edge
reject / externalProtect
```

## 后台预唤醒

### 现有车辆围栏

- 圆心为 HTTP 车辆位置；
- 进入只启动 BLE 警戒；
- 离开只降低活跃度；
- 围栏本身不直接锁解。

### 停车位置备用唤醒（默认关闭）

- 需要 Always 定位；
- 进入后台时优先固定车辆位置作为停车点；
- 车辆位置不可用时，只有手机已在车旁完成 BLE 鉴权才采用手机位置；
- 一次后台会话固定一个停车点，不随手机移动；
- 必须先离开停车区域，再重新进入，才预唤醒 BLE；
- 显著位置变化只处理“由远到近”，不在远离时唤醒；
- 只启动扫描/连接/鉴权，不直接控车。

### pending-connect

生命周期：

```text
已创建 → 扫描中 → 连接中 → 安全重鉴权中 → 已鉴权
                                   └→ 失败/等待系统机会
```

日志记录每个阶段与累计耗时。

### CoreBluetooth 状态恢复

系统恢复 `CBPeripheral` 时：

1. 清空认证随机数、控制密钥、特征就绪态和 pending control；
2. 重新发现服务/特征；
3. 重新执行完整鉴权；
4. 只有再次进入 authenticated 才允许控车。
